VISUAL MAP INTERPRETATION:

- **OCCUPIED CELLS (Colored/Marked):** These represent obstacles OR `existing_locations`. **STRICTLY FORBIDDEN** to build on these specific indices.
- **VOID CELLS (Black/Empty):** These are free spaces. Build here.
- **TARGET CELLS:** Marked coordinates in the input. These are the mandatory connection points for your new geometry.

DATA PROCESSING & GENERATION LOGIC:
1.  **Analyze Target Logic:**
	- **Single Target:** Treat the lone `target_cell` as a "Seed". Grow the location outwards from this point according to the size/shape in the `description`. Generated location or group always include ALL target_cells.a
	- **Multiple Targets:** Your primary goal is to form a coherent structure that **physically connects** all provided `target_cells`.
		- If the description implies a "Path" or "Hall", build a direct connection.
		- If the description implies a "Room" or "Area", use the targets as boundary markers or key points within the volume.

2.  **Context Alignment (Smart Connections):**
	- Review `existing_locations_context` associated with the `target_cells`.
	- Look for functional keywords: "door", "gate", "open wall", "end of corridor".
	- **Directionality:** If a target is a "Door" of an existing house, generate the new topology moving *away* from that door (normal to the surface), ensuring the passage is accessible.

3.  **Collision Avoidance:**
	- Do NOT generate indices that exist in `existing_locations_context`.
	- You may (and often must) be *adjacent* to them to form connections, but never overlap.

INSTRUCTIONS:

	- **Continuity:** Ensure the generated cells form a single connected component (unless description specifies disjoint zones) that links the targets.
3.  **Write Descriptions:**
