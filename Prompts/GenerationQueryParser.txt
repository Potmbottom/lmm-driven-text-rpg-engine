TASK:
Analyze the user's input and extract the intent for the Generation Tool.

INPUT: 
User natural language text.

OUTPUT:
Return a JSON object with the following structure:
{{
    "type": "string", // Options: "location", "group", "object"
    "description": "string", // Comprehensive description of WHAT to generate
    "target_cells": ["string"] or empty array, // Array of "X:Y:Z" coordinates. Examples: ["1:1:0"] or ["1:1:0", "1:2:0"] for connections
    "id": int, // Optional: The ID of the PARENT location to add to. (e.g. "add to location 5"). MAX 1.
    "target_locations": [int] or empty array // Optional: IDs of locations to connect or reference (e.g. "connect 5 and 6").
}}

GUIDELINES:
1. Type Selection:
   - "location": Creating a new distinct place (house, dungeon, clearing).
   - "group": Adding a room or area to an EXISTING location.
   - "object": Creating items, furniture, or NPCs.
2. Description: Keep all adjectives and details.
3. Coordinates (target_cells): 
   - Extract explicitly mentioned coordinates into the list.
   - If the user implies "here" or "current", return an empty list [] (system handles context).
   - If multiple cells are mentioned (e.g., for a path or connection), include all of them.
4. IDs vs Target Locations:
   - field "id": Use this ONLY for hierarchy/attachment. It defines WHERE the new content belongs (e.g. "add a table to room 10"). strictly single value.
   - field "target_locations": Use this for topology/interaction. It defines which EXISTING locations are involved or connected (e.g. "corridor between location 5 and 6").
