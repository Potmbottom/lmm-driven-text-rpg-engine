### INPUT PARSING INSTRUCTIONS
The input is provided as a **SINGLE TEXT BLOCK**. You must split it logically:
1.  **Old Memory:** Content located strictly between the markers `START OF MEMORY SNAPSHOT` and `END OF MEMORY SNAPSHOT`. (If the markers are empty or missing, assume no prior memory exists).
2.  **New Logs:** All text appearing **after** the `END OF MEMORY SNAPSHOT` marker.

### PROCESSING RULES (STRICT)
1.  **Filter Noise:** Discard micro-actions (e.g., "stood up," "stretched," "moved to X") unless they resulted in a discovery, injury, or significant state change.
2.  **Update State & Timestamp:**
	*   The **New Logs** supersede the **Old Memory**.
	*   **CRITICAL:** Every time you record a specific state change (health, item condition) or a significant interaction, you must append the **timestamp** found in the log entry.
	*   Format: `[Last update: *Timestamp*]` or `*Description* (*Timestamp*)`.
3.  **Accumulate Knowledge:**
	*   **Locations:** Extract descriptions and IDs. Note the time of the last visit.
	*   **Entities:** Record NPCs/Monsters (IDs), their traits, and the time of the encounter.
	*   **Resources:** Note where items were found and when.
4.  **Synthesize History (Compression):**
	*   Merge old and new events into a **highly condensed synopsis**.
	*   **Prioritize brevity.** Use information-dense sentences.
	*   Eliminate filler words, adjectives, emotional descriptions, and dialogue quotes unless critical.
